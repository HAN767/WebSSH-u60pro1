# U60 Pro 机身屏幕 UI 设计

## 设计目标

这套 UI 面向 U60 Pro 机身小屏，不做 Web 后台的缩小版。它的核心任务是：

1. 常驻显示最关键的设备状态。
2. 通过三页横向滑动完成高频网络和系统操作。
3. 在息屏、锁屏、点亮、解锁之间保持状态栏可见，减少盲操作。

设计主线：状态栏常驻、首页看信号、第二页管网络、第三页管系统。

## 设备与输入约束

基于 `u60pro-devui-main/examples/go/main.go` 当前测试代码：

- 渲染方式：Go 直接写 DRM/KMS RGB565 framebuffer。
- 刷新方式：整屏 `DIRTYFB`，约 30fps 上限。
- 触摸方式：evdev，支持按下、移动、抬起。
- 坐标处理：显示和触摸均已按设备 180 度安装方向修正。
- UI 建议：固定像素布局、大触控目标、低频状态刷新、少长文本输入。

触控目标建议：

- 最小点击目标：44 x 44 px。
- 开关/按钮高度：48-60 px。
- 列表项高度：44-56 px。
- 页面左右安全边距：8-12 px。
- 顶部状态栏高度：屏幕高度的 10%-13%。

## 总体结构

```text
┌────────────────────────────────┐
│ 时间  网速  网络  信号  电池    │  常驻状态栏
├────────────────────────────────┤
│                                │
│          当前页面内容           │  三页横向滑动
│                                │
└────────────────────────────────┘
```

页面顺序：

1. 首页：当前信号信息 + 当前载波信息。
2. 网络页：Wi-Fi 开关、已连接设备、网络制式切换、锁频、恢复默认。
3. 系统页：屏幕设置、系统信息、设备标识、锁屏、ADB 调试。

交互：

- 左右滑动切换三页。
- 页面底部用 3 个小圆点表示当前位置。
- 不设置底部导航，避免挤占小屏内容空间。
- 所有危险操作使用确认弹窗。

## 常驻状态栏

状态栏从左到右固定为：

1. 时间
2. 实时网速，单位为 `M` / `K` / `b`
3. 网络状态：`5G` / `5GA` / `4G` / `3G`
4. 信号强度 bar
5. 电池图标和百分比

示意：

```text
┌────────────────────────────────┐
│ 10:32  2.4M/s  5GA  ▂▄▆█  ▰ 86% │
└────────────────────────────────┘
```

### 时间

- 格式：`HH:mm`。
- 秒不显示，减少重绘压力。
- 每分钟刷新一次；如果状态栏整体已经被网速刷新带动，可以顺带更新。

### 网速

显示实时下行优先；如果空间足够，可以上下行轮播。

单位规则：

- `< 1024 b/s`：显示 `860b/s`
- `< 1024 K/s`：显示 `238K/s`
- `>= 1024 K/s`：显示 `2.4M/s`

建议 1 秒刷新一次。状态栏空间紧张时只显示一个方向：

- 默认：下行速度。
- 上传明显高于下载时：显示上传速度并加上上箭头图标。

### 网络状态

显示规则：

- `5GA`：5G NSA 或 5G 聚合态，具体定义由设备接口返回值确认。
- `5G`：5G SA 或普通 5G。
- `4G`：LTE。
- `3G`：UMTS / WCDMA / 其他 3G。
- 无服务：显示 `--` 或 `无服务`，优先短文本。

如果后端能区分 SA/NSA，正文页显示完整文案，状态栏保持短标签。

### 信号强度 bar

使用 4 或 5 格信号条，优先根据 RSRP/SINR 综合判断。

建议映射：

| 档位 | 5G/4G RSRP 参考 | 显示 |
| --- | --- | --- |
| 0 | 无服务或未知 | 空 |
| 1 | `< -115 dBm` | 1 格 |
| 2 | `-115 ~ -105 dBm` | 2 格 |
| 3 | `-105 ~ -95 dBm` | 3 格 |
| 4 | `> -95 dBm` | 4 格 |

### 电池

- 显示电池轮廓图标 + 百分比。
- 充电时图标内显示闪电或在百分比前加 `+`。
- 低电量 `< 20%` 使用黄色。
- 极低电量 `< 10%` 使用红色。

## 首页：信号与载波

目标：点亮屏幕后第一眼看清当前移动网络质量。

布局示意：

```text
┌────────────────────────────────┐
│ 10:32  2.4M/s  5GA  ▂▄▆█  ▰ 86% │
├────────────────────────────────┤
│ 5G NSA        n78              │
│ 中国移动      B3 + n78         │
├────────────────────────────────┤
│ RSRP  -84 dBm    SINR   18 dB  │
│ RSRQ  -9 dB      RSSI  -61 dBm │
├────────────────────────────────┤
│ PCI 123         Cell 456789    │
│ TAC 2103        EARFCN 640000  │
├────────────────────────────────┤
│ 当前载波                        │
│ 4G B3 20MHz  +  5G n78 100MHz  │
│ ● ○ ○                          │
└────────────────────────────────┘
```

首页内容：

- 当前网络模式：5G SA / 5G NSA / 5G-A / 4G / 3G。
- 运营商。
- 主频段和聚合频段。
- RSRP、SINR、RSRQ、RSSI。
- PCI、Cell ID、TAC、EARFCN/NRARFCN。
- 当前载波信息：4G/5G 频段、带宽、PCC/SCC 标记。

视觉重点：

- 网络模式和主频段使用最大字号。
- RSRP 和 SINR 作为二级重点。
- 载波明细用较小字号，超过一行时横向截断或进入详情。

点击行为：

- 点击信号大卡：进入“信号详情”弹层。
- 点击载波区域：展开全部载波列表。
- 下拉或点击刷新图标：刷新当前网络信息。

## 第二页：网络控制

目标：完成现场最常用的网络操作。

布局建议用纵向滚动，因为这一页功能多。

```text
┌────────────────────────────────┐
│ 10:32  2.4M/s  5GA  ▂▄▆█  ▰ 86% │
├────────────────────────────────┤
│ Wi-Fi                          │
│ 2.4G  [开关]    5G  [开关]     │
├────────────────────────────────┤
│ 已连接设备  8 台      [查看]    │
├────────────────────────────────┤
│ 网络制式                        │
│ [自动] [5GSA] [5GNSA] [4G] [3G] │
├────────────────────────────────┤
│ 锁频                            │
│ [5G 锁频] [4G 锁频]             │
│ [恢复默认]                      │
│ ● ○ ○                          │
└────────────────────────────────┘
```

### Wi-Fi 开关

控件：

- `2.4G Wi-Fi` 开关。
- `5G Wi-Fi` 开关。

状态：

- 开启：绿色。
- 关闭：灰色。
- 应用中：显示 loading。
- 失败：短提示，保留原状态。

接口映射：

- `GET /api/wifi/settings`
- `POST /api/wifi/state/set`
- 必要时使用 `POST /api/wifi/settings`

### 已连接设备

显示：

- 总数：例如 `已连接设备 8 台`。
- 分组摘要：`无线 6 / 有线 2`。

点击后弹层：

```text
┌──────────────────────────────┐
│ 已连接设备                   │
│ iPhone      -52dBm  866M     │
│ laptop      -61dBm  433M     │
│ 192.168.1.22       有线       │
│ [关闭]                       │
└──────────────────────────────┘
```

列表字段：

- 设备名或 IP。
- MAC 尾号，可选。
- 无线信号。
- 协商速率。
- 有线/无线标记。

### 网络制式切换

固定 5 个按钮：

- `自动`
- `5G SA`
- `5G NSA`
- `4G`
- `3G`

其中“自动”对应设备在 5G/4G/3G 之间自动选择。

交互：

- 点击某个制式先选中，不立即应用。
- 点击“应用”后二次确认。
- 应用后显示“切换中”，10-20 秒内刷新网络状态。

接口映射：

- `POST /api/network/mode`

### 锁频

按钮：

- `5G 锁频`
- `4G 锁频`
- `恢复默认`

5G 锁频弹层：

```text
┌──────────────────────────────┐
│ 5G 锁频                      │
│ 当前 n78                     │
│ [n1] [n28] [n41] [n78]       │
│ [取消]              [应用]   │
└──────────────────────────────┘
```

4G 锁频弹层：

```text
┌──────────────────────────────┐
│ 4G 锁频                      │
│ 当前 B3                      │
│ [B1] [B3] [B5] [B8] [B40]    │
│ [取消]              [应用]   │
└──────────────────────────────┘
```

恢复默认：

- 清除 5G 锁频。
- 清除 4G 锁频。
- 清除小区锁定，如果后端把小区锁定也视为默认配置的一部分。
- 恢复网络制式为自动。

危险提示：

- 锁频和恢复默认都需要确认。
- 确认文案要直接说明“可能短暂断网”。

接口映射：

- `POST /api/network/band/nr`
- `POST /api/network/band/lte`
- `POST /api/network/cell/nr`
- `POST /api/network/cell/lte`
- `POST /api/network/mode`

## 第三页：系统与屏幕

目标：控制屏幕行为、查看系统状态、管理调试能力。

布局建议分为三个区块：屏幕、系统信息、调试/安全。

```text
┌────────────────────────────────┐
│ 10:32  2.4M/s  5GA  ▂▄▆█  ▰ 86% │
├────────────────────────────────┤
│ 屏幕                            │
│ 亮度  [------●---]  70%         │
│ 息屏  [15s] [30s] [1m] [5m]     │
├────────────────────────────────┤
│ 系统                            │
│ CPU 32%  58℃    MEM 46%        │
│ 电池 34℃  4.08V  -420mA        │
│ 充电 5.02V  980mA              │
├────────────────────────────────┤
│ IMEI  86************12          │
│ 版本  vX.Y.Z                    │
│ 锁屏 [开关] 双击点亮 [开关] ADB │
│ ○ ○ ●                          │
└────────────────────────────────┘
```

### 屏幕亮度

控件：

- 滑条：0%-100%。
- 推荐最小亮度限制：10%，避免调到几乎不可见。

交互：

- 拖动时实时预览。
- 抬手后保存。
- 保存失败时回滚到上一次亮度。

### 自动息屏时间

选项建议：

- `15s`
- `30s`
- `1m`
- `5m`
- `从不`

如果锁屏开启，息屏后再次点亮进入锁屏壁纸态。

### 系统信息

你要求显示的字段：

- CPU 占用。
- CPU 温度。
- 内存占用。
- 电池温度。
- 充电电压。
- 充电电流。
- 电池电压。
- 电池电流。
- IMEI。
- 版本号。

刷新建议：

- CPU/内存：2 秒。
- 温度/电压/电流：3-5 秒。
- IMEI/版本号：启动时读取，手动刷新即可。

字段显示建议：

- CPU 占用：`CPU 32%`
- CPU 温度：`58℃`
- 内存占用：`MEM 46%`
- 电池温度：`BAT 34℃`
- 充电电压/电流：`CHG 5.02V 980mA`
- 电池电压/电流：`BAT 4.08V -420mA`
- IMEI 默认脱敏显示，点击详情或长按后显示完整值。

### 锁屏开关

开关：

- `锁屏`：开启/关闭。

规则：

- 关闭锁屏：点亮后直接回到上次页面或首页。
- 开启锁屏：息屏后下次点亮显示锁屏壁纸。
- 解锁成功后进入首页，不恢复到第二/第三页，减少误触后停留在危险操作页。

### 双击点亮

开关：

- `双击点亮`：开启/关闭。

建议：

- 只在息屏态监听双击。
- 双击时间窗口：250-450ms。
- 双击后如果锁屏开启，进入锁屏壁纸；否则进入首页。

### ADB 调试

按钮：

- `开启 ADB`

状态：

- 未知。
- 已开启。
- 开启失败。

确认文案：

```text
开启 ADB 调试？
调试端口可能暴露，请确认当前网络可信。
```

接口映射：

- `POST /api/openadb`

## 锁屏与息屏

### 状态机

```text
正常显示
  -> 到达自动息屏时间
  -> 息屏
  -> 单击电源键或双击点亮
  -> 如果锁屏关闭：进入首页
  -> 如果锁屏开启：进入锁屏壁纸
  -> 长按屏幕
  -> 解锁进入首页
```

### 锁屏壁纸态

锁屏壁纸要求：

- 使用自定义壁纸作为背景。
- 壁纸上方保留常驻状态栏。
- 壁纸下方靠上一点显示一行文字：`长按屏幕解锁`。
- 解锁后进入首页。

布局：

```text
┌────────────────────────────────┐
│ 10:32  2.4M/s  5GA  ▂▄▆█  ▰ 86% │
├────────────────────────────────┤
│                                │
│                                │
│          自定义锁屏壁纸         │
│                                │
│        长按屏幕解锁             │
│                                │
└────────────────────────────────┘
```

长按解锁：

- 长按时长：建议 900-1200ms。
- 按住时显示环形或横条进度。
- 手指提前抬起则取消。
- 解锁后进入首页。

壁纸处理：

- 推荐使用 RGB565 友好的预处理图片。
- 启动时加载并缩放/裁剪到屏幕内容区。
- 状态栏区域不要被壁纸覆盖。
- 如果壁纸不存在，使用默认深色背景和设备名称。

### 息屏态

息屏态应该尽量减少刷新：

- 关闭背光或降到 0。
- 停止页面内容重绘。
- 保留必要输入监听：电源键、双击点亮。
- 网速/系统信息采样可以降频或暂停。

## 视觉规范

推荐色彩：

- 背景：`#15161A`
- 面板：`#20232A`
- 主文字：`#F5F7FA`
- 次文字：`#AEB6C2`
- 主操作：`#4F8BFF`
- 正常：`#46C46F`
- 警告：`#FFD24A`
- 危险：`#E0533A`

字体与排版：

- 状态栏使用紧凑等宽数字。
- 首页网络模式和主频段使用最大字号。
- 系统信息使用两列或三列密集布局。
- 中文标题 15-18 px。
- 正文 12-15 px。
- 重要数字 22-32 px。

小屏原则：

- 状态值比说明文字更重要。
- 每个页面只保留一个主视觉重点。
- 危险按钮使用红色并放在确认弹窗内。
- 不依赖颜色单独表达状态，颜色必须配合文字或图标。

## Go 实现建议

建议从当前 demo 抽出四层：

1. `display`：framebuffer、RGB565、绘制 primitives、flush。
2. `input`：evdev touch、tap/swipe/long press/double tap 识别。
3. `ui`：状态栏、三页路由、弹层、列表、滑条、开关。
4. `data`：本机状态采样、WebSSH API、sysfs/procfs 读取。

核心结构草案：

```go
type App struct {
    Display *Display
    Input   <-chan InputEvent
    UI      *UIState
    Store   *DeviceStore
}

type UIState struct {
    Mode        ScreenMode
    Page        int
    LockEnabled bool
    DimTimeout  time.Duration
}

type ScreenMode int

const (
    ModeNormal ScreenMode = iota
    ModeDimmed
    ModeLockscreen
)
```

事件建议：

```go
type InputEvent struct {
    Type InputType
    X, Y int
    DX, DY int
    Duration time.Duration
}
```

需要识别：

- Tap。
- Horizontal swipe。
- Vertical scroll。
- Long press。
- Double tap when dimmed。

刷新策略：

- 状态栏网速：1 秒。
- 首页信号：2-5 秒。
- 第二页 Wi-Fi/设备列表：5 秒，弹层打开时可 1-2 秒。
- 第三页 CPU/内存：2 秒。
- 电池/温度：3-5 秒。
- 静止页面不需要 30fps 重绘，只有动画/滑动/进度时提高刷新。

## MVP 范围

建议第一版实现：

1. 常驻状态栏：时间、网速、网络状态、信号、电池。
2. 三页横向滑动和页面圆点。
3. 首页：信号信息和载波信息。
4. 第二页：2.4G/5G Wi-Fi 开关、已连接设备数量、网络制式切换、5G/4G 锁频入口、恢复默认。
5. 第三页：亮度、息屏时间、系统信息、IMEI/版本号、锁屏、双击点亮、ADB。
6. 锁屏壁纸态：状态栏 + 自定义壁纸 + `长按屏幕解锁` + 解锁进入首页。

第二版再补：

- 已连接设备详情列表。
- 5G/4G 锁频多选弹层。
- 状态栏上下行网速切换。
- 壁纸管理和预览。

实现前还需要确认：

1. 屏幕真实分辨率。
2. 可用字体和中文渲染方案。
3. 亮度 sysfs 节点路径。
4. 电池、充电、电池温度、CPU 温度节点路径。
5. 设备网络状态里 `5GA` 的精确定义。
6. 锁屏壁纸文件存放路径和图片格式。
7. WebSSH 后端是否给本机屏幕 UI 提供免登录本地 API，或使用固定本地 token。
